# 开场
Bright老师您好, 我是Shuhan。之前和您邮件联系的UCL学生，这段时间我重新把之前的Demo打磨了一下，今天特来像您汇报。就像我提到的，我是 Uhomes 的忠实用户。在使用 UhomesAI 的过程中，我切实感受到了 AI 给租房带来的便利。这也激发了我作为一个开发者的好奇心：现有的 AI 主要解决了‘信息检索’的问题，但如果我们在现有的基础上，引入 Agentic Workflow（代理工作流），是不是能让 AI 进一步进化，去主动处理像‘多维度比价’、‘实时通勤计算’这样更复杂的决策任务？

抱着这个想法，我实现了当前的系统，主要是想探索一下 Agentic Workflow在留学租房场景里的应用潜力。下面我以此模拟一个 UCL 新生的视角，展示一下它的工作流程。

# Prompt 1:

## Prompt:
你好，我是今年9月要去 UCL (University College London) 读研的新生。我目前对伦敦租房市场一无所知。 请帮我查一下 2025年最新的市场数据：如果是 UCL 步行 20 分钟以内的区域，现在的 En-suite（独立卫浴）平均租金大概是多少钱一周？如果我的预算只有 £200/周，我是不是必须住得远一点？

## 旁白：
第一个场景是模拟新生。很多新生的预算其实是不切实际的。 如果Agent直接去数据库里硬搜很可能搜不到结果，所以他先启动了一个**‘市场调研’的动作。 它调用了网络搜索，查到了 Bloomsbury 的最新行情，然后像一个学长一样告诉我：‘在这个预算下，你可能需要用通勤时间换居住空间’。 这个步骤可以把用户引导到一个合理的决策区间。

# Prompt 2:

## Prompt:
明白了，看来我得接受通勤。 请帮我在 Zone 2 或 Zone 3 的范围里，寻找符合以下条件的公寓：

预算在 £350 - £450 /周。

到 UCL (Euston Square/Warren Street 站) 的通勤时间控制在 30分钟以内（请帮我实际计算一下）。

必须是 PBSA (留学生公寓)，不要私人房东。 请给我推荐 2-3 个具体的公寓名字，并告诉我它们的租金和通勤时间。

## 旁白：
现在模型将会根据我们的要求，自动的通过我们内置的推荐算法提供几个房源推荐。筛选出了预算内且满足 30 分钟通勤圈的 5 套房源，并直接在右侧面板展示了列表。

# Prompt 3:

## Prompt:
我看你推荐的这套房子似乎性价比不错。我们就以此为例。 我是一个女生，比较在意安全。请不要只告诉我‘公寓有门禁’这种套话。我想知道这个公寓所在的 具体社区（Neighborhood）的真实治安数据 是怎么样的？晚上从地铁站走回公寓的那几百米路，根据历史犯罪数据看，是否安全？

## 旁白：
现在我们以女生租客的视角，来考察一下社区的安全性。 这个时候，Agent 并没有给出一个笼统的“安全”评价，而是触发了一个**‘外部数据分析’的工具**。它调用了 UK Police API 获取了该街区最近 6 个月的真实犯罪数据（10起），甚至根据我们预设的算法计算出了一个 95/100 的安全评分。它通过数据分析告诉用户：虽然整体安全，但要注意具体的犯罪类型（如盗窃）。这种基于数据的理性评估，比单纯的文字描述更有说服力。

# Prompt 4:

## Prompt:
好的，关于这个房源，还有一个决定性的问题： 我非常依赖自己做饭，必须买中国调料。请精确计算一下：从这个公寓的大门口步行到最近的一家“中国超市”（Chinese Supermarket，不仅是便利店），到底有多远？请给出精确的店名、米数和步行时间

## 旁白：
这个场景，是关于生活便利性的问题。 这里，Agent 结合地图数据，精确计算了 POI（兴趣点）。它不仅列出了周边的 Waitrose 和 Tesco，还精准定位了距离仅 28 米的中餐厅和 130 米的 China City。 这对于习惯自己做饭的留学生来说，是一个非常实用的信息。

# Prompt 5:

## Prompt:
如果我把 Scape Bloomsbury 和另一个更靠近市中心但贵很多的 City 放在一起比。 考虑到我刚才强调的‘在意晚归安全’和‘做饭需求’，作为 AI 顾问，你觉得我为了省钱选前者是理性的吗？ 请帮我权衡一下利弊，给我一个最终建议。

## 旁白：
现在我们进入了深度推理（Reasoning）阶段。Agent 敏锐地捕捉到了 Scape Bloomsbury 低价背后的隐性条件——它是 Twin Room（双人间）。它不仅仅在比较价格，更是在对比“生活方式”：它警告用户双人间意味着牺牲隐私和极其严格的访客政策（不能过夜）。这展示了 Agent 具备了语义理解和顾问思维，它能指出用户可能忽略的合同细节风险。

# Prompt 6:

## Prompt:
我觉得我可能接受不了Twin ensuite，你是否能帮我横向对比一下Scape Bloomsbury和City的所有成本。通勤+租房费用，周边的基础设施（有中国超市，中国餐厅额外加分）然后再给我提供一个更完整的建议。

## 旁白：
最后，我们让agent帮忙做一个更全面的成本效益分析。 它会把租金、通勤费用和生活便利性等多个因素结合起来，给出一个更完整的建议，帮助用户做出最终决策。 虽然在最终建议的逻辑上它还需要进一步微调（比如更好地平衡用户对 Twin Room 的排斥），但这种将“隐性交通成本”货币化并叠加到房租上进行横向对比的能力，展示了Agent在处理复杂、多维度问题上的强大能力。
